La mise en page (layouts)

Mise en page définie en XML

Déclarée dans res/layout/mon_layout.xml

Utilise différentes propriétés XML

On peut utiliser un éditeur visuel

Chargement dans le code de l'activité avec setContentView(R.layout.mon_layout)

Avantage : séparation du traitement (Java) et de la présentation (XML)

Désavantage : interface utilisateur statique

Mise en page définie par programmation (Java)

Instantiation d'un layout, définitions de propriétés, insertions de "sous-layouts" (comme en XML)

Pas besoin d'un fichier de layout (tout est dans le code Java)

LinearLayout fenetre = new LinearLayout(this);
fenetre.setUnAttribut(...);
fenetre.setUnAutreAttribut(...);
fenetre.addView(widget ou layout);
...
setContentView(fenetre);

Avantage : interface utilisateur dynamique

Désavantage : le code mélange traitement et présentation

Attributs XML

Chaque classe de Layout a une class interne nommée LayoutParams qui définit les paramètres généraux du Layout

Ces paramètres sont tous appelés android:layout_xxx et se rapportent généralement aux dimensions et aux marges

Les classes de Layout elles-mêmes définissent des attributs plus spécifiques (dont les noms ne débutent pas par android:layout)

Exemple :

<LinearLayout
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:gravity="center_horizontal"
   android:background="@color/couleur_1">
   ...
<LinearLayout>

Taille

On trouve d'abord les attributs android:layout_height (hauteur) et android:layout_width (largeur)

La valeur match_parent remplit l'espace disponble dans le parent, moins le "padding"

Tandis que la valeur wrap_content utilise la taille naturelle, plus le "padding"

L'attribut android:layout_weight permet de spécifier des tailles proportionnelles (détails plus loin)

Alignement

L'attribut android:layout_gravity spécifie comment la vue ou le layout est disposé à l'intérieur de son parent

L'attribut android:gravity spécifie comment le contenu (ex : texte) est disposé à l'intérieur même de la vue

Valeurs possibles :

top, bottom, left, right, center_vertical, center_horizontal, center (vertical et horizontal), fill_vertical, fill_horizontal, fill (vertical et horizontal), clip_vertical (?), clip_horizontal (?)

Marges

C'est l'espace vide à l'extérieur du composant

Les valeurs possibles sont :

android:layout_marginTop, android:layout_marginBottom, android:layout_marginLeft, android:layout_marginRigh

L'unité recommandée pour exprimer la dimension des formes géométriques est le dp :

dp density-independent pixels (s'ajuste à la résolution du périphérique)

Tandis que l'unité recommandée pour exprimer la dimension des caractères est le sp :

sp scaled pixels (s'ajuste selon la taille de police préférée)

Mais il est toujours possibles d'utiliser dans tous les cas les unités suivantes :

ptpoints
pxpixels
ininches (pouces)
mm millimètres

Remplissage (padding)

Espace vide à l'intérieur du composant (entre les contours et le contenu)

Les valeurs possibles sont :

android:paddingTop, android:paddingBottom, android:paddingLeft, android:paddingRight

Les unités sont les mêmes que pour les marges

Autres attributs importants

Identificateur

L'attribut android:id est utilisé si le code Java a besoin d'une référence à une vue

Aussi utilisé en XML avec le RelativeLayout

Couleurs

L'attribut android:background spécifie la couleur d'arrière-plan d'un layout, exemple : android:background="@color/rose_bonbon"

Aussi pour les images d'arrière-plan, exemple : android:background="@drawable/singe"

L'attribut android:textColor est utilisé pour les composants comme les zones de texte (TextView) et les boutons (Button)

Il existe plusieurs façons de représenter une couleur : #rrggbb, #aarrggbb, @color/nom_couleur

Gestionnaire de clic

Attribut android:onClick

Spécifie une méthode "public" de l'activité principale qui prend un paramètre de type View (le composant cliqué) et ne retourne rien (void)

La propriété "layout_weight"

En utilisant cette propriété, la taille d'un composant devient proportionnelle à son poids (weight)

D'abord, on donne au composant une hauteur nulle, exemple : android:layout_height="0dp"

Ensuite, on utilise des poids relatifs pour chacun des composants

Par exemple, trois composants placés dans le même conteneur et ayant des poids relatifs de 1, 1 et 2 occuperont respectivement 25%, 25% et 50% de l'espace du conteneur

L'approche suggérée est de faire en sorte que la somme des poids soit égale à 100 et de traiter les valeurs comme des pourcentages

Voici un exemple :

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">    
	
    <TextView
        android:layout_width="match_parent"
        android:background="#ffcccc"
        android:layout_height="0dp"
        android:layout_weight="25"
        android:text="layout_weight=25"
        android:textSize="20sp"/>
        
    <TextView
        android:layout_width="match_parent"
        android:background="#ccffcc"
        android:layout_height="0dp"
        android:layout_weight="25"
        android:text="layout_weight=25"
        android:textSize="20sp"/>	
        
    <TextView
        android:layout_width="match_parent"
        android:background="#ccccff"	    
        android:layout_height="0dp"
        android:layout_weight="50"
        android:text="layout_weight=50"
        android:textSize="20sp"/>	
	
</LinearLayout>

TableLayout

Utilise un genre de tableau (sans bordures) pour disposer les composants (et les layouts imbriqués)

Comme dans le cas des tableaux HTML, les numéros de rangées et de colonnes sont déterminées automatiquement et non spécifiés de manière explicite

Quelques attributs importants de TableLayout

L'attribut android:stretchColumns est un index (ou une liste d'index) pour spécifier la ou les colonnes qui doivent être étirées si le tableau est plus petit que son parent

Contrepartie de l'attribut précédent, android:shrinkColumns spécifie la ou les colonnes qui doivent être rétrécies si le tableau est plus grand que son parent

L'attribut android:collapseColumns indique la ou les colonnes qui ne seront pas affichées (remarque : l'affichage peut être rétabli plus tard par programmation)

Attributs importants d'un élément dans un TableRow

Les composants sont placés principalement à l'aide de la balise <TableRow>

Les éléments sont normalement disposés de gauche à droite, sans avoir besoin de spécifier le numéro de la colonne

L'attribut android:layout_column permet de spécifier exactement la colonne à utiliser, laissant vides les colonnes précédentes

L'attribut android:layout_span indique le nombre de colonnes que l'élément doit chevaucher (équivalent du "colspan" en HTML)

Remarque : il n'y a pas d'équivalent du "rowspan" de HTML (la solution est d'utiliser des tableaux imbriqués)

Un exemple :

<TableLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:stretchColumns="1">

    <TableRow>
        <Button android:text="Btn 0"/>
        <Button android:text="Btn 1"/>
        <Button android:text="Btn 2"/>
    </TableRow>
    <TableRow>
        <Button android:text="Btn 3"/>
        <Button android:text="Btn 4"/>
        <Button android:text="Btn 5"/>
    </TableRow>	
    <TableRow>
        <Button android:text="Btn 6"/>
        <Button android:text="Btn 7" android:layout_column="2"/>
    </TableRow>
    <TableRow>
        <Button android:text="Btn 8"/>
        <Button android:text="Btn 9" android:layout_span="2"/>
    </TableRow>

</TableLayout>

RelativeLayout

Pas très populaire auprès des débutants parce que le plus compliqué de tous

Mais aussi le moins gourmand en ressources



Nooo! Not the Relative Layooout!


Dans ce type de layout, on place les composants les uns par rapport aux autres

Les composants sont identifiés à l'aide de la propriété android:id

Les attributs d'alignement

Alignement à l'intérieur du conteneur (layout) :

android:layout_alignParentTop
android:layout_alignParentRight
android:layout_alignParentBottom
android:layout_alignParentLeft

android:layout_centerHorizontal
android:layout_centerVertical
android:layout_centerInParent

Tous ces attributs prennent les valeurs true ou false

Alignement dans l'espace d'un autre composant :

android:layout_alignTop
android:layout_alignRight
android:layout_alignBottom
android:layout_alignLeft

Ces attributs prennent comme valeur l'identificateur d'un composant

Alignement par rapport à un autre composant :

android:layout_toLeftOf
android:layout_toRightOf
android:layout_above
android:layout_below

Ces attributs prennent comme valeur l'identificateur d'un composant

Premier composant : 

<Button id="@+id/bouton_1"
    android:layout_alignParentRight="true"
    .../>

Deuxième composant : 

<Button
    android:layout_toLeftOf="@id/bouton_1"
    .../>

Remarque : on utilise "@id" et non "@+id", puisque le composant a déjà été déclaré plus haut dans le fichier
